Standalone VS Modular Architecture

Ok so there is 2 ways / 2 architectures that are used for building Angular applicatons. One is the Modular architecture and one is the Standalone architecture. I feel even though the Modular architecture is a bit outdated, it is the best way to learn Angular and going forward, it won't be too hard to switch to the Standalone structure. So i will be going forward with the modular architecture and if you want to know more about the standalone architecture, you can go here.

This is the basic Module Architecture:

Here, the features you wanna implement in angular are divided into Modules (also called as feature-modules) and the each part of the feature inside that as component. You can build the app the exact way you want it when you use the Module structure. Now when you're doing in the modular structure, the root module/main module is the file named 'app.module.ts'. So when i say root/main module that's the file i am refering to.

Now generally to start an angular application you would use:

            
              ng new my-angular-app

But from Angular 17 onwards, this starts up a standalone application, to do in modular format, you need to start application with:

            
              ng new my-angular-app --no-standalone
