Angular Components

Components are the basic building blocks of an Angular application. Everything you code will be mostly inside a component. Say, you wanna make an About page, you will generate a component named about, add the content you need and in that component itself you will handle the necessary functions needed for that component. Let's dig a little deeper.

Let us start by generating a component. To generate a component, use :

            
              ng generate component component-name

Now once you have generated this, you get a component as a folder with 4 parts/files in it :

  • your-component.component.css
  • your-component.component.html
  • your-component.component.spec.ts
  • your-component.component.ts
  • your-component.component.css

    • Here, you can add the css elements you need for your component. Anything css you need for that specific components other than inline css like tailwind css goes here. This part of the structure is similar to the styles.css file you make in a regular html/css project.

    Example :

  • your-component.component.html

    • This is basically what the user see when they are viewing that component. You can do it the way you have learnt HTML and have some extra angular features as well. You dont have to include the whole typical HTML format, you only need the body part of it. This part is purely for the HTML code you need to implement in the application. This is prolly the part that made ya'll smile :)

    Example :

  • your-component.component.spec.ts

    • Maybe i'm saying this as a novice but don't touch this file. I really don't have much idea of what it does other than something for testing and as beginners i recommend you don't go digging much deep into it either right now. A basic idea of what it does is ,It sets up the testing environment, initializes the component, and contains test cases to verify that the component's methods, properties, and template work as expected. This helps ensure the component behaves correctly and integrates properly with other parts of the application.

    So basically, nothing you need to worry about right now XD

  • your-component.component.ts

    • Now this is a really important file . This is the main part of your component. If you want, you have an option to not use the css and html file while writing all its code inside this. The main use of this file is to define all your required functions for the component, initalising constructors and many other things like using ngOnInit which i'm pretty sure you won't understand right now. Just know that this is where all the logic required for your component goes. You will get to know more about this when you further dive deep into Angular.
